## Короткий вывод

Проект уже выглядит как рабочий прототип: есть React/Vite-интерфейс, PHP API для авторизации, гостевая главная, новости, профиль, поиск, адаптивное меню, организация, админки, педагогический кабинет.

Но по ТЗ проект пока не готов как полноценная информационная система. Главная проблема: большая часть модулей работает как фронтенд-демо через `localStorage` и статические массивы, а ТЗ требует полноценную базу данных, API, роли, права, документы, редакционные процессы, официальные источники Республики Беларусь и реальные пользовательские данные.

## Что сделано нами в этой ветке работ

### Гостевой режим и лендинг
- Главная страница работает как гостевая лендинговая страница.
- Гость видит описание системы, общие новости, часть базы знаний и ссылку на правовой раздел.
- Приватные разделы больше не перебрасывают гостя сразу на `/login`; вместо этого показывается приветственная плашка с кнопками входа и регистрации.
- При выходе пользователь остается на главной странице, без жесткого перехода на login/register.

### Новости
- Создана отдельная страница `/news`.
- Добавлена кнопка перехода к новостям в хедер и меню.
- Новость открывается как отдельная страница `/news/:newsId`.
- У публикаций есть дата, автор, теги, категория, специализация, аудитория, источники.
- Добавлена роль `Редактор`.
- Редактор, администратор организации и глобальный администратор могут создавать, редактировать и удалять новости.
- Новости разделены на публичные и доступные после входа.
- Добавлен блок рекомендаций новостей с учетом специализации пользователя и статуса `Новичок` / `Опытный`.
- Из `content.md` добавлено наполнение: 78 публикаций, часть открыта гостям, часть только после входа.
- Создан промпт для сбора достоверной информации и источников: `guidelines/news-content-research-prompt.md`.

### Поиск
- Главный поиск доработан в стиле поисковой строки: при вводе появляются предложения.
- Предложения собираются из модулей проекта, статей, новостей и популярных тем.
- Поиск ведет к наиболее подходящему результату или на `/knowledge?q=...`.

### Адаптив
- Для desktop сохранено верхнее меню.
- Для tablet добавлено левое меню, которое можно скрывать.
- Для mobile меню перенесено вниз, используются иконки с короткими подписями.
- Иконка профиля на mobile перенесена в хедер справа напротив логотипа.
- По нажатию на мобильную иконку профиля открывается список: профиль, настройки, выход. Для гостя: вход и регистрация.

### Onboarding
- Шаги onboarding визуально переработаны: круги с номерами, ровная линия между шагами, активное состояние.

## Рекомендуемый порядок доработки

1. Сначала исправить юридическую основу под РБ: трудоустройство и НПА.
2. Затем перенести данные из `localStorage` в MySQL/API.
3. После этого сделать RBAC и панель прав.
4. Потом добить новости/базу знаний как CMS с модерацией.
5. Затем полноценно реализовать организации и документы.
6. Потом центр инцидентов с локальными инструкциями.
7. В конце довести педкабинет, экспорт, Groq, безопасность и performance.
